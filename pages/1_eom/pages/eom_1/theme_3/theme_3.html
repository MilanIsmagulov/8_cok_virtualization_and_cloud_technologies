Виды виртуализации

Существует несколько видов виртуализации, каждый из которых предназначен для разных целей и областей применения:

1. Симуляция, она же полная эмуляция. При этом виде виртуализации виртуальная машина эмулирует все устройства, как будто она настоящая физическая машина. При этом операционная система внутри виртуальной машины остается неизменной. Это позволяет эмулировать различные аппаратные архитектуры, то есть создавать виртуальные среды, которые ведут себя как компьютеры с разным оборудованием.

2. Нативная виртуализация, также известная как частичная эмуляция представляет собой метод виртуализации, при котором не все компоненты системы виртуализируются, а только те, которые могут работать в изолированном режиме. Это позволяет включать гостевое оборудование в архитектуру хоста, что дает большую гибкость и удобство использования.

Основная идея нативной виртуализации заключается в том, что гостевая операционная система работает непосредственно на аппаратном обеспечении хоста, без необходимости эмуляции аппаратных ресурсов. Это достигается путем предоставления гостевой ОС прямого доступа к физическим ресурсам, таким как процессор, память и периферийные устройства. Таким образом, гостевая ОС может выполняться с полной производительностью, как если бы она работала на физическом сервере.

3. Паравиртуализация – это вид виртуализации, при котором гостевые операционные системы знают о своем виртуальном окружении и взаимодействуют с гипервизором для более эффективного выполнения задач. При паравиртуализации гостевые ОС адаптированы к виртуальной среде.

4. Виртуализация на уровне ОС – это метод, который позволяет создавать несколько независимых виртуальных серверов на одном физическом компьютере. Суть данного подхода заключается в том, что различные виртуальные системы работают параллельно на одной операционной системе хоста. Этот подход широко используется для предоставления услуг хостинга, когда несколько пользователей нуждаются в своих собственных виртуальных серверах, и все они могут работать на одном ядре физического сервера.

5. Виртуализация на уровне приложений. В данном подходе вместо использования отдельных виртуальных машин для каждого приложения, у нас есть контейнер, в котором содержится всё необходимое для запуска приложения. Это означает, что приложение можно запустить на любой совместимой платформе без необходимости устанавливать его заново

6. Виртуализация ресурсов – это важная часть управления различными видами ресурсов, такими как хранилище данных, сеть, пространства имен и другие. Этот подход позволяет упростить процесс управления ресурсами путем их объединения, концентрации и совместного использования. Виртуализацию ресурсов можно реализовать двумя способами. В первом случае создаются отдельные наборы ресурсов для пользователей. Во втором случае большой ресурс делится на меньшие однотипные части. Оба подхода делают использование ресурсов более удобным для пользователей
